## 잡담
오늘은 기능을 크게 새로 만들기보다는,
앞으로 계속 굴릴 수 있는 작업 환경을 정리한 날이었다.

보안, 계정, 배포 신뢰도처럼
눈에 덜 보이지만 오래 가는 바닥 작업을 많이 했다.

## 로그 기준 기간
- KST 기준 `2026-02-17 06:00` ~ `2026-02-18 05:59`
- 실제 사용자 요청이 확인된 구간: `2026-02-17 09:10` ~ `2026-02-17 18:39`

## 오늘 한 큰 일들
1. GitHub OAuth 유출 대응 흐름 정리
- 한 일: 토큰 노출 여부 확인, 노출 토큰의 계정 소속 확인, 히스토리 정리 필요성 점검.
- 왜: 푸시/배포를 계속하려면 먼저 계정 보안 리스크를 닫아야 해서.

2. 블로그 게시 시간(KST) 정합성 점검
- 한 일: 배포 후 게시 시각이 한국 시간 기준으로 어긋나 보이는 문제 확인 및 교정 흐름 정리.
- 왜: 기록형 블로그에서는 시간 메타데이터 신뢰도가 생각보다 중요해서.

3. GitHub 재연결 가이드 정리
- 한 일: 기존 인증 정리 -> 새 인증 등록 -> 로그인 상태 검증까지 한 번에 정리.
- 왜: "당장 접속"보다 "다음에도 안 꼬이는 운영 방식"이 필요해서.

4. 멀티 계정 SSH 분리 방식 확정
- 한 일: 계정명을 `robotjuhwan`/``으로 확정하고, SSH 별칭 기반 분리 등록 절차로 정리.
- 왜: 저장소마다 다른 계정으로 실수 푸시하는 문제를 막기 위해.

5. 출력 형식 제약 실험(JSON only)
- 한 일: 짧은 테스트 입력으로 응답 포맷 강제 조건이 안정적으로 동작하는지 확인.
- 왜: 자동화 파이프라인에서 예측 가능한 출력 형태가 필요해서.

## 오늘 결론
오늘의 핵심은 "새 기능 추가"보다
"보안/계정/배포를 안정적으로 운영할 기반 정리"였다.

앞으로 코덱스 작업을 계속 밀어붙이기 위해
필수적인 운영 체계를 다진 하루였다.
